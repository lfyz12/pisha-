# Спецификация: знакомство с инструментами и анимация нажатия кнопок

## Цель

Добавить в интерфейс ПИША контекстное знакомство с инструментами для новых пользователей и анимацию Material-riple при нажатии на кнопки. Стек должен быть поднят для проверки результата.

## Субъект и аудитория

ПИША — система рейтинга студентов Уральской инженерной школы УрФУ. Пользователи:

- **Студенты** — смотрят рейтинг, стипендии, аналитику, общаются с ИИ-ассистентом.
- **Администраторы** — загружают данные из Excel, управляют политиками доступа, начисляют баллы, настраивают ИИ-правила.

Задача страницы/функции — помочь новому пользователю быстро понять, что он может делать в текущем разделе, не блокируя рабочий процесс.

## Дизайн-решения

### Токены

| Роль | Значение | Примечание |
|------|----------|------------|
| `accent` | `#dd5e27` | Существующий брендовый терракотовый. Используется для акцента в подсказках, кольца выделения, рипла. |
| `accent-soft` | `#fff3ed` | Фон активного пункта меню и лёгких акцентов. |
| `overlay` | `rgba(15, 23, 42, 0.22)` | Затемнение страницы при spotlight. Достаточно заметное, но не перекрывающее контент. |
| `ring` | `#dd5e27` | Кольцо вокруг выделенного элемента. |
| `shadow-spotlight` | `rgba(221, 94, 39, 0.25)` | Тень под выделенным элементом, приподнимающая его. |
| `surface` | `#ffffff` | Карточка подсказки и основные поверхности. |
| `text-body` | `#444240` | Основной текст подсказки. |

### Типографика

Используем существующие шрифты проекта (Tailwind-стек). Подсказка должна читаться как системная подсказка, а не как маркетинговый блок:

- Заголовок карточки: `font-weight: 600`, `14px`.
- Текст подсказки: `13px`, `line-height: 1.6`.
- Акцентные слова-якоря: `font-weight: 600`, цвет `accent`, подчёркивание пунктиром.

### Layout-концепция

Контекстная подсказка — карточка в правом нижнем углу страницы (`position: fixed` или `absolute` в рамках layout). Не перекрывает основной рабочий столбец. Содержит 1–2 предложения, кнопки «Понятно» и «Скрыть», и якорные слова.

Кнопка повторного вызова подсказок — плавающая круглая кнопка с вопросительным знаком в левом нижнем углу. Появляется только если пользователь хотя бы раз закрыл подсказки.

### Signature-элементы

**1. Hover-якорь с затемнением и дрожанием связанного элемента.**

Когда пользователь наводит на выделенное слово в подсказке, страница затемняется, связанный элемент обводится кольцом цвета акцента, получает мягкую тень и слегка дрожит. Это делает onboarding похожим на инженерный указатель: «вот этот конкретный инструмент сейчас в фокусе».

**2. Прыгающая оранжевая точка-проводник.**

При запуске тура по странице или приложению из кнопки «?» маленькая оранжевая точка вылетает из кнопки и движется к целевому элементу по плавной траектории. При подлёте точка увеличивается, растворяется и превращается в рамку-спотлайт вокруг элемента. После подтверждения пользователя рамка сжимается обратно в точку, и та прыгает к следующему элементу.

Это два заметных, но согласованных эффекта; остальной интерфейс остаётся спокойным.

### Почему это не дефолт

Три типичных AI-дефолта для такой задачи:
1. Тёплый кремовый фон + сериф + терракотовый акцент.
2. Тёмный фон + кислотный акцент.
3. Газетная раскладка с тонкими линиями.

Мы избегаем их: используем существующий бело-терракотовый стиль проекта, не добавляем лишних декоративных элементов, не делаем общий модальный тур. Вместо скучного пошагового мастера в окне добавляем живую точку-проводник, которая физически ведёт взгляд к инструменту. Spotlight + tremor + прыгающая точка — осознанный риск, привязанный к теме «точного указания на инструмент».

## Пользовательский сценарий

### Первый вход

1. Пользователь входит в систему.
2. Попадает на дашборд. Подсказка не показывается на дашборде сразу — сначала пользователь видит интерфейс.
3. При первом переходе в раздел (например, «Рейтинг») появляется контекстная подсказка внизу справа.
4. Подсказка объясняет 1–2 инструмента текущей страницы через якорные слова.
5. Пользователь может:
   - Навести на якорное слово и увидеть spotlight + tremor связанного элемента.
   - Нажать «Понятно» — подсказка больше не появляется для этой страницы.
   - Нажать «Скрыть» — подсказка исчезает, но может быть вызвана кнопкой «?».
6. При посещении других страниц появляются свои подсказки (только при первом визите).

### Повторный вызов

- Плавающая кнопка «?» в левом нижнем углу открывает меню со списком доступных обучающих режимов:
  - «Подсказки этой страницы» — показывает контекстную карточку текущей страницы.
  - «Тур по странице» — запускает точку-проводник по якорям текущей страницы.
  - «Тур по приложению» — запускает точку-проводник по ключевым разделам меню.

## Режим тура с точкой-проводником

### Визуальное описание

- **Точка**: оранжевый круг `12px`, цвет `accent`, с мягкой тенью.
- **Движение**: при переходе к следующему элементу точка движется по кривой Безье (CSS `offset-path` или расчёт через JS), слегка подпрыгивая по пути.
- **Приближение**: за последние `200ms` точка увеличивается с `12px` до `40px`, opacity падает с `1` до `0`.
- **Растворение в рамку**: одновременно с исчезновением точки вокруг целевого элемента появляется спотлайт-рамка (то же кольцо, что и при hover-якоре).
- **Обратная трансформация**: при нажатии «Далее» рамка сжимается в точку, и та летит к следующему элементу.

### Режимы

**1. Тур по текущей странице**

- Доступен всегда через меню кнопки «?».
- Точка обходит все якорные элементы текущей страницы в порядке чтения (слева направо, сверху вниз).
- Каждый шаг показывает короткий текст-бабл рядом с целевым элементом.
- Последний шаг предлагает закрыть тур или перейти к туру по приложению.

**2. Тур по приложению**

- Запускается автоматически при первом входе студента или администратора.
- Можно вызвать повторно через меню кнопки «?».
- Точка последовательно подсвечивает пункты бокового меню: Дашборд → Рейтинг → Стипендии → Аналитика → ИИ-чат → Профиль.
- При переходе между страницами точка «перепрыгивает» через layout: исчезает в текущем меню и появляется у целевого пункта на новой странице.
- Для администратора маршрут включает административные разделы: Импорт → Политики → Начисление → ИИ-правила.

### Управление туром

- **«Далее» / «Пропустить»** — в бабле у текущего элемента.
- **Esc** — прерывает тур и возвращает интерфейс в обычное состояние.
- **Клик вне бабла** — не прерывает тур, чтобы случайное нажатие не сбивало обучение.

### Технические детали

- Тур не полагается на координаты элементов вручную: каждый шаг ссылается на `data-onboarding-id` целевого элемента.
- Если элемент не найден в DOM (например, скрыт фильтром), шаг пропускается автоматически.
- Переходы между страницами в туре по приложению используют `setTimeout`/`requestAnimationFrame` после `navigate`, чтобы дождаться рендера новой страницы.

## Анимация нажатия кнопок

Все кнопки системы получают Material-riple:

- При `pointerdown` создаётся круг, растущий от точки касания до краёв кнопки.
- Цвет рипла: белый с прозрачностью `0.25` для `default` варианта, чёрный с прозрачностью `0.1` для `outline`/`ghost`.
- Длительность: `500ms`.
- Easing: `cubic-bezier(0.4, 0, 0.2, 1)`.
- Рипл удаляется после окончания анимации.
- Должна быть версия без анимации для `prefers-reduced-motion`.

## Состояние

- Хранение в `localStorage` в одном ключе `pisha-onboarding`, значением которого является объект, сгруппированный по `userId`:
  ```ts
  {
    [userId: string]: {
      dismissed: string[]; // ключи страниц, закрытых навсегда через «Понятно»
      hidden: string[];    // ключи страниц, скрытых через «Скрыть», но доступных через «?»
    }
  }
  ```
- При смене пользователя или роли состояние читается для текущего `userId`; если записи нет — подсказки показываются заново.
- Бэкенд не изменяется.
- Приватность: не сохраняет персональные данные, только `userId` и идентификаторы подсказок.

## Доступность

- Подсказка не блокирует фокус (`aria-live="polite"`).
- Кнопки «Понятно» и «Скрыть» имеют читаемые label.
- Якорные слова — интерактивные элементы с `role="button"` и `tabIndex=0`, поддержка `Enter`/`Space`.
- Spotlight при фокусе через клавиатуру работает так же, как при наведении.
- Тур с точкой:
  - Бабл шага имеет `role="dialog"` и `aria-modal="false"`.
  - Фокус перемещается на кнопку «Далее» в бабле.
  - Экранный диктор зачитывает заголовок и описание шага.
- `prefers-reduced-motion`:
  - Отключает tremor.
  - Отключает ripple.
  - Spotlight появляется мгновенно без анимации затемнения.
  - Тур с точкой заменяется на последовательность статических спотлайтов без движущейся точки.

## Реализация с имеющимися библиотеками

В проекте уже есть React 19, TypeScript, TailwindCSS v4, shadcn/ui (Radix) и Zustand. Новые анимационные библиотеки не добавляются.

- **Material-riple**: реализуется внутри обёртки над `Button` через DOM-элемент `span`, создаваемый в момент `pointerdown`. Рост круга — CSS `@keyframes` с изменением `scale` и `opacity`. Удаление — по событию `animationend`.
- **Spotlight и tremor**: оверлей и вырез — абсолютно позиционированные `div` с inline-стилями, вычисленными через `getBoundingClientRect()` целевого элемента. Тремор — CSS `@keyframes` с `translateX`/`rotate`. Для cutout используется `box-shadow: 0 0 0 9999px ...`.
- **Точка-проводник**: движение рассчитывается через `requestAnimationFrame` и CSS-переменные `--x`, `--y`, либо через CSS `offset-path` для простых траекторий. Морфинг в рамку — анимация `width`, `height`, `border-radius`, `background`, `border` одного и того же элемента.
- **Позиционирование бабла**: `getBoundingClientRect()` целевого элемента + простая логика «справа/снизу, если помещается, иначе слева/сверху».
- **Состояние**: Zustand для runtime-состояния (активный тур, текущий шаг) + `localStorage` для персистентности.
- **Иконки**: существующий компонент `Icon` из `components/ui/icon.tsx` для кнопки «?».

## Открытые решения

1. **Показывать ли подсказку сразу или после небольшой задержки?**
   - Решение: задержка `400ms` после монтирования страницы, чтобы интерфейс успел отрисоваться.

2. **Показывать ли подсказку на дашборде?**
   - Решение: на дашборде не показывать — там много информации, лучше дать пользователю осмотреться, а знакомить уже в конкретных разделах.

3. **Что делать при смене роли?**
   - Решение: состояние привязано к `userId`, поэтому при смене роли подсказки покажутся заново для новой роли.

- [ ] Подсказки появляются при первом визите на страницы для студента и администратора.
- [ ] Якорные слова при наведении затемняют страницу и выделяют связанный элемент.
- [ ] Связанный элемент слегка дрожит при наведении.

- [ ] Рамка тура точно совпадает с границами целевого элемента (допуск ±2px), не перекрывает его и не смещена.

- [ ] Работает `prefers-reduced-motion`.
- [ ] Нет регрессий в TypeScript (`npm run typecheck`) и линтере (`npm run lint`).
